# STM32 Logic Analyzer Learning

## Hardware

- STM32F401RE Nucleo Board
- 24 MHz 8-Channel USB Logic Analyzer

## Project Included

### UART_ANALYSIS

Capture and analyze UART communication generated by STM32 using a Logic Analyzer and PulseView.

The project verifies UART transmission by observing the actual TX waveform and decoding transmitted data using the PulseView UART protocol decoder.

## Peripherals Used

- USART1
- GPIO
- Logic Analyzer

## Concepts Learned

- Logic Analyzer Setup
- PulseView Usage
- UART Signal Analysis
- UART Frame Structure
- Start Bit Detection
- Stop Bit Detection
- LSB First Transmission
- UART Protocol Decoding
- STM32 Alternate Function Configuration
- Hardware-Level Debugging

## Experiments Performed

### Experiment 1

Transmit:

```c
printf("U");
```

Purpose:

- Verify UART frame structure
- Observe Start Bit, Data Bits and Stop Bit
- Verify UART Decoder Operation

Observed Output:

```text
55
```

ASCII Equivalent:

```text
U
```

---

### Experiment 2

Transmit:

```c
printf("HELLO\r\n");
```

Observed Output:

```text
HELLO
```

This confirms successful UART transmission and decoding using the Logic Analyzer.

## UART Frame Structure

```text
Idle  Start  D0 D1 D2 D3 D4 D5 D6 D7  Stop
HIGH   LOW
```

For character 'U' (0x55):

```text
0x55 = 01010101

LSB First:

1 0 1 0 1 0 1 0
```

## Debugging Notes

During initial testing, USART2 (PA2) was monitored.

Since PA2 is connected internally to the Nucleo ST-Link Virtual COM Port, direct observation was not straightforward.

The experiment was moved to USART1 (PA9), allowing direct monitoring of the UART TX signal using the Logic Analyzer.

This helped in understanding:

- STM32 Alternate Functions
- UART Signal Generation
- Logic Analyzer Connections
- Protocol Decoding
